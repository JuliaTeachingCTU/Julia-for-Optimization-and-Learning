Other useful packages

Distributions.jl

julia> using Distributions

julia> D = Normal(2, 0.5)
Distributions.Normal{Float64}(μ=2.0, σ=0.5)

julia> x = rand(D)
1.9303246148776716
julia> using StatsPlots

julia> x = rand(D, 10000);

julia> histogram(x; label = "empirical density function", normalize = :pdf)
Plot{Plots.GRBackend() n=1}

julia> plot!(D; label = "true density function", linewidth = 2)
Plot{Plots.GRBackend() n=2}
julia> Dfit = fit(Normal, x)
Distributions.Normal{Float64}(μ=1.9994476975556932, σ=0.5058721507477312)

julia> histogram(x; label = "empirical density function", normalize = :pdf)
Plot{Plots.GRBackend() n=1}

julia> plot!(D; label = "$D", linewidth = 2)
Plot{Plots.GRBackend() n=2}

julia> plot!(Dfit; label = "$Dfit", linewidth = 2)
Plot{Plots.GRBackend() n=3}

BSON.jl

The BSON is a package for working with the Binary JSON serialisation format. It can be used as a general store for Julia data structures.

julia> using BSON

julia> BSON.bson("test.bson", Dict(:a => [1+2im, 3+4im], :b => "Hello, World!"))

julia> BSON.load("test.bson")
Dict{Symbol,Any} with 2 entries:
  :a => Complex{Int64}[1+2im, 3+4im]
  :b => "Hello, World!"
julia> using BSON

julia> BSON.bson("test.bson", a = [1+2im, 3+4im], b = "Hello, World!")

julia> BSON.load("test.bson")
Dict{Symbol,Any} with 2 entries:
  :a => Complex{Int64}[1+2im, 3+4im]
  :b => "Hello, World!"
julia> using BSON: @save, @load

julia> a = [1+2im, 3+4im];

julia> b = "Hello, World!";

julia> @save "test.bson" a b # Same as above

julia> @load "test.bson" a b # Loads `a` and `b` back into the workspace

ProgressMeter.jl

The ProgressMeter package provides excellent utilities for printing progress bars for long-running computation. The package provides @showprogress macro, that can be used for printing progress bar for for loops in the following way

julia> using ProgressMeter

julia> @showprogress 1 "Computing..." for i in 1:50
           sleep(0.1)
       end
Computing... 20%|███████▊                               |  ETA: 0:00:04

The same syntax can be used also with map/pmap/reduce function. Progress bars can be also created manually, which allows additional formatting of the output. For example, it is possible to print and update information related to the computation by using the showvalues keyword as follows

julia> x, n = 1 , 10;

julia> p = Progress(n);

julia> for iter in 1:10
           x *= 2
           sleep(0.5)
           ProgressMeter.next!(p; showvalues = [(:iter, iter), (:x, x)])
       end
Progress: 100%|█████████████████████████████████████████| Time: 0:00:10
  iter:  10
  x:     1024

BenchmarkTools.jl

The BenchmarkTools package provides a framework for writing and running groups of benchmarks as well as comparing benchmark results. The primary macro provided by BenchmarkTools is @benchmark.

julia> using BenchmarkTools

julia> @benchmark sin(x) setup=(x=rand())
BenchmarkTools.Trial:
  memory estimate:  0 bytes
  allocs estimate:  0
  --------------
  minimum time:     5.600 ns (0.00% GC)
  median time:      6.500 ns (0.00% GC)
  mean time:        8.411 ns (0.00% GC)
  maximum time:     833.763 ns (0.00% GC)
  --------------
  samples:          10000
  evals/sample:     1000

The setup expression in the example above, is run once per sample, and is not included in the timing results. Another handy macro provided by the package is the @btime macro. The output of this macro is similar to the built-in @time macro

julia> A = rand(3,3);

julia> @btime inv($A);
  830.964 ns (4 allocations: 1.95 KiB)

Note, that we use $ to interpolate variable A into the benchmark expression. Any expression that is interpolated in such a way into the benchmark expression, is "pre-computed" before benchmarking begins.
